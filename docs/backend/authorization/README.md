# PrintForge — Integration Authorization

## Overview

- Each integration has a unique **signing secret**
- Stored in two places: WordPress (`wp_options`) and the PrintForge database
- Never exposed to the browser
- Used for two purposes only: authorizing designer requests and authorizing webhook calls

---

## 1. Designer Authorization

**Setup:**
- When WordPress renders the page, PHP uses the signing secret to generate a short-lived signed token
- The token is injected into the designer iframe URL at render time
- The token is tied to a specific product ID and expires after 1 hour

**On Add to Cart:**
- The designer sends the token with the presign request to Fastify
- Fastify decodes the token and checks it has not expired
- Fastify looks up the correct secret using the product ID
- Fastify recomputes the expected signature and compares it against the token
- If they match → request is authorized and a presigned URL is issued

**Key properties:**
- WordPress signs independently, Fastify verifies independently — no round trip between them
- A captured token cannot be reused for a different product
- Token becomes useless after expiry

---

## 2. Webhook Authorization

**On order completion:**
- WooCommerce signs the webhook payload using the integration secret
- The signature is included in the request header
- Fastify receives the webhook and looks up the secret for that integration
- Fastify recomputes the expected signature from the raw payload and compares it against the header
- If they match → webhook is legitimate and processing continues

---

## 3. Secret Lifecycle

**Onboarding:**
- Print shop installs the WordPress plugin
- Signing secret is generated by the platform and provided to the print shop
- Print shop enters the secret in the plugin settings
- Same secret is stored in the PrintForge database under the integration record

**Rotation:**
- New secret is generated on the platform side
- Plugin settings are updated manually on the WordPress side
- Tokens issued before rotation will fail — acceptable given the 1-hour expiry window

---

## Summary

| Purpose | Who signs | Who verifies |
|---|---|---|
| Designer presign request | WordPress | Fastify |
| WooCommerce webhook | WooCommerce | Fastify |

- One secret per integration
- The secret never reaches the browser
